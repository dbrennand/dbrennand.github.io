Managing Secrets in Ansible 🔑

As with most automation, we need to use credentials to authenticate to our servers and other applications. Examples of secrets include usernames and passwords, API keys, SSH keys, etc. When using these types of secrets in playbooks, we need to store them securely but also still allow Ansible to access them when needed.

Introducing Ansible Vault 🔐

Ansible Vault is a command line tool that is part of the ansible-core package and allows us to encrypt secrets in our playbooks, inventory and vars files. It can encrypt an entire file or a specific variable within a file. Furthermore, Ansible Vault uses the AES256 encryption algorithm to encrypt files and variables using a password provided by the user.

Using Ansible Vault

Let’s take a look at how we can use Ansible Vault to encrypt a file.

Encrypting a File 🔒

To encrypt a file use the ansible-vault encrypt command, providing the path for the file to be encrypted. When prompted enter a password to encrypt the file:

1ansible-vault encrypt vars/main.yml
2New Vault password:
3Confirm New Vault password:
4Encryption successful

Now we can see the file is encrypted:

Editing or Viewing an Encrypted File

To edit or view the contents of an encrypted file, use the ansible-vault edit or ansible-vault view command respectively:

1ansible-vault edit vars/main.yml
2Vault password:
3ansible-vault view vars/main.yml
4Vault password:

Decrypting a File 🔓

To decrypt the file, use the ansible-vault decrypt command and as before, provide the path to the file and the password:

1ansible-vault decrypt vars/main.yml
2Vault password:

Next, let’s see how we can use Ansible Vault to encrypt a variable.

Encrypting a Variable

To encrypt a variable, use the ansible-vault encrypt_string command providing the name of the variable and the value to encrypt. When prompted, enter a password to encrypt the variable:

 1ansible-vault encrypt_string -n my_secret_var "<Secret to encrypt here>"

Great! So we now know how to encrypt a file and a variable, but how about a real world example! 🌎

Real World Example

I sometimes use Hetzner Cloud to spin up virtual servers for testing. In this short playbook which I use to create the server, there is a variable called hetzner_api_key which contains my API key. I don’t want to store this API key in plain text in my playbook! 😱 🙅‍♂️

 1# playbook.yml
 2---
 3- name: Provision Hetzner VPS
 4  hosts: all
 5  gather_facts: false
 6  vars:
 7    hetzner_api_key: "my-secret-api-key"
 8  tasks:
 9    - name: Create a basic server
10      hetzner.hcloud.hcloud_server:
11        api_token: "{{ hetzner_api_key }}"
12        location: fsn1
13        name: my-server
14        server_type: cx11
15        image: ubuntu-18.04
16        state: present

1ansible-playbook -c local -i localhost, --ask-vault-pass playbook.yml

Prompting for the Vault Password Non-Interactively

The above example was interactively prompting for the vault password. So how do we do this non-interactively? Say we want to run the playbook in a CI/CD pipeline. We can’t prompt for the password in this type of scenario 🤔

To solve this, we can use a vault password file.

First, add the password to a file called .vault_password and lock down the permissions::

1# Note the space at the beginning of the command!
2# This is needed to prevent the password from being stored in the shell history.
3 echo "my-encryption-password" > .vault_password
4chmod 600 .vault_password

Now, we run the playbook with the --vault-password-file option instead:

1ansible-playbook -c local -i localhost, --vault-password-file .vault_password playbook.yml

Conclusion

In this post we have covered how to use Ansible Vault for safely storing secrets for use in our playbooks. We’ve covered encrypting files and variables and how to use a vault password file to prompt for the vault password non-interactively.
