Interacting with the ECS S3 API using the aws-sdk/client-s3 package

07 Dec, 2021

Recently at work, a customer raised a ticket about programmatically interacting with UKCloud’s Cloud Storage service.

UKCloud’s Cloud Storage service is an object storage solution based on Dell EMC Elastic Cloud Storage (ECS). Access is via a RESTful API, which also provides support for Amazon’s S3 API.

UKCloud: Getting Started Guide for Cloud Storage 1

This ticket was interesting as the customer was using the @aws-sdk/client-s3 JavaScript package to upload images to the service. Prior to this ticket, I hadn’t used this package before or have much experience with S3 object storage and programmatically interacting with it. In this blog post, I want to share my learnings and provide a couple of examples in Node.js for interacting with UKCloud’s Cloud Storage service using this package.

[NOTE]

It is worth noting that the code examples in this post should work with any S3 object storage provider, just modify the endpoint and region!

S3 Terminology

Before we get started, it’s important to understand certain terminology around Amazon’s S3 API.

Objects

An object is a file and any metadata that describes that file. Objects consist of object data and metadata. The metadata is a set of name-value pairs that describe the object. These pairs include some default metadata, such as the date last modified … You can also specify custom metadata at the time that the object is stored.

Amazon Web Services: What is Amazon S3? 2

Amazon Web Services: Uploading, downloading, and working with objects in Amazon S3 3

Object Keys 🔑

An object key (or key name) is a unique identifier of an object.

Amazon Web Services: Creating object key names 4

An example of an object key could be Testing/Requirements.pdf or Accounting/Payslips.xls. Note that the object keys are prefixed with a directory.

Buckets

A bucket is a container for objects… Every object is contained in a bucket.

Amazon Web Services: What is Amazon S3? 2

A bucket has a flat structure with no actual concept of directories however, you can prefix object keys with (sub)directories to create a directory structure 📁

Now that’s out of the way, let’s begin interacting with the ECS S3 API using the @aws-sdk/client-s3 for JavaScript 😎

Node.js Examples 👨‍💻

The following packages are used in these examples:

  • @aws-sdk/client-s3 - Documentation.

  • @aws-sdk/s3-request-presigner - Documentation.

  • fs

Begin by creating two files: examples.mjs and dockerfile. Then, paste the snippet below into the dockerfile:

FROM node:16-alpine
RUN mkdir -p /usr/src/app && chown -R node:node /usr/src/app
WORKDIR /usr/src/app
RUN npm install @aws-sdk/client-s3@3.43.0 @aws-sdk/s3-request-presigner@3.44.0
COPY --chown=node:node examples.mjs examples.mjs
USER node
CMD ["node", "examples.mjs"]

[NOTE] Place the Node.js examples below into examples.mjs.

Initialise the S3Client

Start by initialising the S3Client by importing the required package:

 1import { S3Client } from '@aws-sdk/client-s3';
 2
 3// Retrieve constants from environment variables
 4const S3_ENDPOINT = process.env.S3_ENDPOINT;
 5const REGION = process.env.REGION;
 6const BUCKET_NAME = process.env.BUCKET_NAME;
 7
 8// Initialise client
 9const client = new S3Client({
10    endpoint: S3_ENDPOINT,
11    region: REGION,
12});

Sending Commands with the S3Client

Now, let’s use the S3Client to create an object in a bucket. For this type of action use the PutObjectCommand:

Create an Object

 1import { S3Client, PutObjectCommand } from '@aws-sdk/client-s3';
 2
 3// ...
 4
 5// Create an object (testObject.txt) prefixed with a directory (test-dir/) in a bucket
 6try {
 7    await client.send(
 8        new PutObjectCommand({
 9            Bucket: BUCKET_NAME,
10            Key: 'test-dir/testObject.txt',
11            Body: 'Uploaded test object.'
12        })
13    );
14    console.log(`Successfully created object 'test-dir/testObject.txt' in the bucket '${BUCKET_NAME}'`);
15} catch (err) {
16    console.error(`An error occurred creating object 'test-dir/testObject.txt' in the bucket '${BUCKET_NAME}': ${err}`);
17};

Let’s see if it works! 😄 Build and run the container image:

 1docker build -t aws-sdk-s3:nodejs .
 2# The aws-sdk looks for the environment variables
 3# AWS_ACCESS_KEY_ID and AWS_SECRET_ACCESS_KEY automatically
 4docker run --rm --name aws-sdk-s3 \
 5    -e S3_ENDPOINT="https://cas.cor00005.ukcloud.com" \
 6    -e REGION="cor00005" \
 7    -e BUCKET_NAME="db-bucket" \
 8    -e AWS_ACCESS_KEY_ID="..." \
 9    -e AWS_SECRET_ACCESS_KEY="..." \
10    aws-sdk-s3:nodejs
11
12# Output

😱 It worked! Congrats! You’ve just created your first object in a bucket! 😎

List and Get Objects

Another action is to list objects in a bucket and get them. Here is an example I wrote for that:

 1import {
 2    S3Client,
 3    PutObjectCommand,
 4    ListObjectsCommand,
 5    GetObjectCommand
 6} from '@aws-sdk/client-s3';
 7
 8// ...
 9
10// List of all objects (max 1000) in a bucket
11try {
12    var objects = await client.send(
13        new ListObjectsCommand({ Bucket: BUCKET_NAME })
14    );
15} catch (err) {
16    console.error(`An error occurred listing objects for the bucket '${BUCKET_NAME}': ${err}`);
17};
18
19// Iterate over objects in the bucket and perform GetObjectCommand
20try {
21    for (const [index, object] of objects.Contents.entries()) {
22        console.log(`${index} - Getting object: ${object.Key}`);
23        const resp = await client.send(
24            new GetObjectCommand({
25                Bucket: BUCKET_NAME,
26                Key: object.Key
27            })
28        );
29        console.log(`${index} - Object: ${object.Key}, ETag: ${resp.ETag}`);
30    };
31} catch (err) {
32    console.error(`An error occurred getting objects for the bucket '${BUCKET_NAME}': ${err}`);
33};

Upload and Download Objects

Uploading and downloading objects is done often, so below are examples I wrote for this:

 1// ...
 2import fs from 'fs';
 3
 4// ...
 5
 6// Create a file to be upload
 7fs.writeFileSync('/usr/src/app/uploadfile.txt', 'Upload me!', (_) => {
 8    console.log("Successfully wrote file '/usr/src/app/uploadfile.txt'");
 9});
10
11// Upload an object (file) to a bucket
12try {
13    // Load the contents of 'uploadfile.txt'
14    fs.readFile('/usr/src/app/uploadfile.txt', 'utf8', async function (_, data) {
15        await client.send(
16            new PutObjectCommand({
17                Bucket: BUCKET_NAME,
18                Key: 'uploadfile.txt',
19                Body: data
20            })
21        )
22        console.log(`Successfully uploaded object '/usr/src/app/uploadfile.txt' to the bucket '${BUCKET_NAME}'`);
23    });
24} catch (err) {
25    console.error(`An error occurred uploading object '/usr/src/app/uploadfile.txt' to the bucket '${BUCKET_NAME}': ${err}`);
26};
27
28// Download an object (file) from a bucket
29try {
30    const downloadFile = await client.send(
31        new GetObjectCommand({
32            Bucket: BUCKET_NAME,
33            Key: 'test-dir/testObject.txt'
34        })
35    );
36    // Create a write stream
37    const writeStream = fs.createWriteStream('/usr/src/app/testObject.txt');
38    // Pipe the object's body to the write stream
39    downloadFile.Body.pipe(writeStream);
40    console.log(`Successfully downloaded object 'testObject.txt' from the bucket '${BUCKET_NAME}'`);
41} catch (err) {
42    console.error(`An error occurred downloading object 'test-dir/testObject.txt' from the bucket '${BUCKET_NAME}': ${err}`);
43};

Generate a Public URL for an Object

The final example I want to share is generating a public URL for an object so it can be downloaded. The example below generates a URL valid for one hour:

 1// ...
 2import { getSignedUrl } from "@aws-sdk/s3-request-presigner";
 3
 4/// ...
 5
 6// Get a public URL for an object in a bucket
 7// Valid for 1 hour
 8try {
 9    const command = new GetObjectCommand({
10        Bucket: BUCKET_NAME,
11        Key: 'uploadfile.txt'
12    });
13    // Get public URL for the object
14    const publicUrl = await getSignedUrl(client, command, { expiresIn: 3600 });
15    console.log(`Successfully generated public URL for object 'test-dir/testObject.txt' in the bucket '${BUCKET_NAME}': ${publicUrl}`);
16} catch (err) {
17    console.error(`An error occurred generating public URL for object 'test-dir/testObject.txt' in the bucket '${BUCKET_NAME}': ${err}`);
18};
